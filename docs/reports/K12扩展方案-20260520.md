### 1.2 代码质量问题

#### 严重程度: HIGH

**H1. `_handle_answer` 中掌握度更新逻辑错误**
- 文件: `src/agent/tutor.py:841-846`
- 问题: 无论答对答错，`new_level` 都设为 `MasteryLevel.EXPOSING`。应使用 `StudentModel.update_mastery()` 的智能升级逻辑，而非每次重置为EXPOSING。
```python
# 当前代码（BUG）
if result.is_correct:
    new_level = MasteryLevel.EXPOSING  # 应该升级
else:
    new_level = MasteryLevel.EXPOSING  # 两个分支一样

# 应该从历史记录推断正确的掌握度
```
- 影响: 学生掌握度永远无法正确升级到MASTERED，学习系统形同虚设。

**H2. XP计算重复且不一致**
- 文件: `src/agent/tutor.py:871-877`
- 问题: `student.total_xp` 通过直接修改ORM对象和 `update_student_xp()` 两种方式同时更新，可能导致XP被加两次。`update_student_xp()` 内部已经是 `+=` 操作，但代码在调用前还手动 `student.total_xp += xp_gain`。
```python
student.total_xp += xp_gain   # 第一次加
await self._store.update_student_xp(student_id, xp_gain)  # 第二次加（内部也是+=）
```

**H3. `asyncio.create_task` 在非async上下文使用**
- 文件: `src/engine/review.py:1975`
- 问题: `_persist_review_schedule` 是同步方法，但调用了 `asyncio.create_task()`。如果不在running event loop中调用会抛异常。复习计划持久化可能静默失败。

**H4. WebSocket `heartbeat_task` 引用在 except 中可能未定义**
- 文件: `src/web/ws.py:2537`
- 问题: 如果 `heartbeat_task` 在 `asyncio.create_task` 之前发生异常（如 `agent.start_session` 失败），`heartbeat_task` 变量未定义，导致 `NameError`。

#### 严重程度: MEDIUM

**M1. JSON解析正则不处理嵌套结构**
- 文件: `src/engine/explain.py:894`, `quiz.py:1038`, `grader.py:1376`
- 问题: 使用 `re.search(r'\{[^{}]*\}', content, re.DOTALL)` 只能匹配单层JSON。如果LLM返回嵌套对象（如 `recommendations` 数组），会匹配失败。
- 已有修复迹象: `diagnose.py:1666` 用了 `r'\{[^{}]*"recommendations".*\}'`，但这也不是通用方案。
- 建议: 使用 `json.JSONDecoder().raw_decode()` 或提取 ```json 代码块。

**M2. 全局单例模式不是线程安全的**
- 文件: 多个模块使用 `global _default_xxx` + `if xxx is None` 模式
- 问题: 在多线程或多协程场景下可能导致竞态条件。虽然Python GIL提供了一定保护，但 `await` 点之间的间隙仍可能出问题。
- 建议: 使用 `asyncio.Lock` 保护初始化。

**M3. prompt模板用字符串替换而非模板引擎**
- 文件: `src/engine/quiz.py:1008`
- 问题: `self._prompt_template.replace('{name}', ...)` 如果 `description` 包含 `{grade}` 等占位符字符，会产生意外替换。
- 建议: 使用 Python `str.format()` 或 Jinja2。

**M4. 掌握度逻辑 `FUZZY` 状态从不降级到 `EXPOSING`**
- 文件: `src/memory/student.py:637-641`
- 问题: `EXPOSING` 状态下，`recent_accuracy < 0.3` 时升级到 `FUZZY`（方向反了），`FUZZY` 状态下 `recent_accuracy < 0.4` 时保持在 `FUZZY`。整个升级路径缺少降级逻辑。

**M5. `datetime.now()` 作为默认值存在时间漂移**
- 文件: `src/memory/student.py:572`, `src/agent/session.py:1484`
- 问题: `field(default_factory=datetime.now)` 用的是dataclass创建时的时间。如果ORM对象被缓存后重用，`updated_at` 可能不准确。

**M6. CORS配置默认允许所有来源**
- 文件: `src/config/settings.py:66-69`
- 问题: 开发和默认环境 `allow_origins=["*"]`，生产环境回退到 `["*"]` 当 `CORS_ORIGINS` 为空时。
- 建议: 生产环境如果未配置CORS_ORIGINS应抛出警告或拒绝启动。

#### 严重程度: LOW

**L1. 重复的 `@pytest.mark.asyncio` 装饰器**
- 文件: `tests/test_store.py:37-38`
- 问题: 同一个测试方法上有两个 `@pytest.mark.asyncio`。

**L2. `student.total_xp` 直接被修改后未commit**
- 文件: `src/agent/tutor.py:873-874`
- 问题: 修改了ORM对象的属性但没有通过session commit，实际值可能没持久化。

**L3. 硬编码版本号 "2.0.0" 出现多处**
- 文件: `src/web/health.py:209,313`, `src/web/app.py:2126`, `src/web/models.py:2068`
- 建议: 从 `src/__init__.py` 的 `__version__` 统一读取。

**L4. `datetime.utcnow()` 已弃用**
- 文件: `src/web/health.py:55,159`
- 警告: `datetime.datetime.utcnow()` 在Python 3.12中已标记为弃用，应使用 `datetime.now(datetime.UTC)`。

**L5. 错误信息泄露内部细节**
- 文件: `src/web/api.py:2600,2625`
- 问题: `str(e)` 直接返回给客户端，可能暴露内部实现。

## 第三部分：K12知识图谱扩展方案

### 3.1 当前数据结构分析

#### 核心数据模型 (`src/knowledge/math_g3g5.py`)

```python
@dataclass
class KnowledgePoint:
    id: str              # "math_g3_001" - 格式: {subject}_g{grade}_{seq}
    name: str            # "乘法口诀"
    grade: int           # 3, 4, 5
    subject: str         # "数学"
    difficulty: Difficulty  # EASY=1..EXPERT=5
    prerequisites: list[str]  # ["math_g3_001"]
    description: str
    examples: list[str]
    common_mistakes: list[str]
```

#### 知识图谱接口 (`src/knowledge/graph.py`)

```python
class KnowledgeGraph:
    add_point(point)                          # 添加知识点
    get_point(point_id) -> KnowledgePoint     # 获取单个
    get_all_points() -> list                  # 获取全部
    get_prerequisites(point_id) -> list       # 直接前置
    get_prerequisites_recursive(point_id)     # 递归前置
    get_dependents(point_id) -> list          # 后续依赖
    get_points_by_grade(grade) -> list        # 按年级过滤
    get_points_by_difficulty(difficulty)       # 按难度过滤
    find_learning_path(target, mastered)       # 学习路径
```

#### 扩展接口分析

**接口兼容性**: 好。`KnowledgeGraph` 接受 `list[KnowledgePoint]` 参数，只需传入更多知识点即可扩展。

**需要修改的地方**:
1. `math_g3g5.py` 中的 `ALL_POINTS` 列表 - 添加新年级数据
2. `get_points_by_grade()` - 已支持任意 `int` grade，无需修改
3. ID命名规范 `math_g{grade}_{seq}` - 需要统一约定

### 3.2 扩展设计原则

1. **每个年级独立文件**: `math_g6.py`, `math_g7.py` 等
2. **ID规范**: `math_g{grade}_{seq:03d}` (如 `math_g6_001`)
3. **前置关系跨年级**: 高年级可依赖低年级知识点
4. **难度重新标定**: 高年级的 EASY 不等于低年级的 EASY，引入"年级×难度"二维调节
5. **向后兼容**: `math_g3g5.py` 保持不变，新文件通过 `__init__.py` 聚合

### 3.4 自适应难度引擎的年级维度扩展

#### 当前机制

当前 `AdaptiveDifficulty` 是纯表现的：根据连续答对/答错调整1-5级难度。**不感知年级差异**。

#### 扩展设计：二维难度矩阵

```
年级难度基线 (Grade Baseline) + 表现难度偏移 (Performance Offset)
```

**实现方案**:

```python
class AdaptiveDifficulty:
    def __init__(self, min_difficulty=1, max_difficulty=5):
        self.min_difficulty = min_difficulty
        self.max_difficulty = max_difficulty
        self._student_states: dict[str, DifficultyState] = {}
        # 新增: 年级难度基线
        self.GRADE_BASELINE = {
            3: 1, 4: 2, 5: 2,   # 小学
            6: 2, 7: 3,          # 初中
            8: 3, 9: 4,          # 初中高年级
        }

    def get_difficulty(self, student_id, history=None,
                       default_difficulty=2, grade=3):
        state = self._get_state(student_id, default_difficulty)
        if history:
            self._update_from_history(state, history)

        # 年级基线 + 表现偏移
        baseline = self.GRADE_BASELINE.get(grade, 2)
        effective = max(self.min_difficulty,
                       min(baseline + (state.current_difficulty - 2),
                           self.max_difficulty))
        return effective
```

**关键改动**:
1. `get_difficulty()` 和 `update_after_answer()` 增加 `grade` 参数
2. 年级基线表定义各年级的起始难度
3. 表现偏移在基线上做 +/- 调整
4. 向后兼容: 不传 `grade` 时使用原逻辑

#### Difficulty 枚举语义细化

```python
class Difficulty(IntEnum):
    EASY = 1        # 直接应用公式
    MEDIUM = 2      # 需要一步推理
    HARD = 3        # 多步推理或综合运用
    VERY_HARD = 4   # 需要建模或抽象
    EXPERT = 5      # 竞赛级或开放性问题
```
